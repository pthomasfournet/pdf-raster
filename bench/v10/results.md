## Bench-gate evaluation

The Phase 10 spec gates the architectural win on three criteria that are measurable on a single dev box (criterion 4 is cross-vendor proof of life and stays out of scope here).

**Criterion 1 — CUDA path no regression vs v0.7.0 mode D (threshold ±5%):**

Baseline live-captured from v0.7.0 binary on the same hardware + corpus subset (`D-v070-baseline.txt`).  We re-bench v0.7.0 rather than comparing against `bench/v070/D.txt` because that file was taken under earlier driver/system state — corpus-02 reported ~210ms there but ~500ms today on the same v0.7.0 binary, so the stale numbers would conflate driver drift with Phase 10 code drift.

| Corpus | v0.7.0 mode D | v10 mode D | Δ | Pass? |
|---|---|---|---|---|
| 01 native text small | 253±12ms | 254±9ms | +0.4% | ✓ |
| 02 native vector+text | 554±41ms | 492±9ms | -11.2% | ✓ |
| 03 native text dense | 834±19ms | 856±11ms | +2.6% | ✓ |
| 04 ebook mixed | 858±32ms | 830±4ms | -3.3% | ✓ |
| 05 academic book | 1322±54ms | 1234±27ms | -6.7% | ✓ |

**Criterion 1: PASS** (regression = >5% slower than baseline)
